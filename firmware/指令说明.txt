上电后程序流程：
	1、连接wifi
	2、连接到TCP服务端
	3、开启透传
	4、发送数据

指令(通过串口向Arduino发送指令，注意选择换行和回车NL&CR都要，或者发送新行)：

	当运行在1、2阶段时，若需要修改WiFi或者TCP服务端，先输入 stop 使其停止尝试连接，该指令输入后需要等待回复，方可进行下面的操作（后续介绍的任何指令都需要等待回复后再进行下一步操作）

	当运行在透传模式下时，若需要修改WiFi或者TCP服务端，先输入 quit 使其退出透传模式，等待成功退出透传模式，方可进行下面的操作。再次输入 quit 可以恢复透传模式

	修改WiFi指令：		设置WiFi名：		set id=xxxx	如：set id=tao
				设置WiFi密码：		set pwd=xxx	如：set pwd=12345678
	修改TCP服务端指令：	设置TCP服务端ip：		set ip=xxx	如：set ip=192.168.197.23
				设置TCP服务端端口：	set port=xxx	如：set port=8088
	修改完以上信息后，若需要下次上电后按照新的信息进行连接，则需要输入 save 指令进行保存，否则下次上电后仍将以旧有的信息进行连接

	在任何状态下，若想使板子从头执行1、2、3、4流程，输入指令 go ahead，例如对连接信息进行修改后，需要重新连接时，可输入该命令。板子将自动退出透传模式，断开原有WiFi连接，转而连接新的WiFi，连接新的TCP服务端，再开启透传模式传输数据

	在任何状态下，设置采样频率：set freq=xxx	如：set freq=100		单位Hz，范围：1~130，若小于1，会自动设置为1，若大于130，会自动设置为130

	在透传模式下，想要暂停数据传输，输入 hold， 想要继续发送数据亦输入 hold
